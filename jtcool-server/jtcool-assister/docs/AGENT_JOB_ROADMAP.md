# Agent 求职导向改造路线图（4周）

## 目标
通过在现有项目中系统实现 Agent 开发能力，产出可量化、可演示、可写简历的成果，提升面试竞争力。

## 第1周：Agent 从“能跑”到“可解释”
### 周目标
实现标准化 Agent Loop：`Plan -> Act -> Observe -> Reflect -> Answer`。

### 交付物
1. `AgentRun` 运行记录结构：`step/tool/latency/token/error`。
2. 执行约束：最大步数、单步超时、失败重试。
3. 每轮输出执行轨迹摘要（不暴露敏感推理内容）。
4. 基础单测：成功路径、超时路径、重试路径。

### 简历亮点
设计并实现多步 Agent 执行引擎，支持失败恢复与执行可观测。

## 第2周：工具治理 + 人在回路
### 周目标
让 Tool 调用具备生产约束和安全机制。

### 交付物
1. Tool 参数 schema 校验（类型/必填）。
2. 超时、熔断、幂等键。
3. 高风险工具审批流：`pending/approved/rejected`。
4. 审计日志（调用人、参数摘要、耗时、结果）。

### 简历亮点
实现 Agent Tool Governance 与 Human-in-the-loop 风险控制流程。

## 第3周：评测体系（核心加分项）
### 周目标
把“主观效果”变成“可量化改进”。

### 交付物
1. 50~100 条离线评测集（政策问答/天气/拒答）。
2. 指标：准确率、引用命中率、工具调用成功率、平均步数、token 成本。
3. 回归脚本：每次改 prompt/策略自动出报告。

### 简历亮点
搭建 Agent Evals 管线，基于指标驱动策略优化。

## 第4周：工程化展示 + 求职包装
### 周目标
把项目升级为可演示的系统化作品。

### 交付物
1. 可观测性：结构化日志 + Trace（建议 OTel）。
2. Docker Compose：一键启动 app + postgres + pgvector。
3. README 升级：架构图、时序图、评测结果、压测数据。
4. 5分钟 Demo 脚本（含失败场景与审批流）。

### 简历亮点
从原型完成到可部署、可观测、可评估的 Agent 系统落地。

## 每周并行输出（求职复利）
1. 每周 1 篇技术复盘（问题、方案、数据、结果）。
2. 每周 1 份高质量 PR 说明（设计、权衡、验证）。
3. 每周沉淀 1 个 STAR 面试案例。
